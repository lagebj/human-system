# Concept Return Log

**Tracks how concepts accumulate meaning across the manuscript.**

Concepts must not restart as if new. When a concept reappears, it must carry the weight of its earlier movements.

---

## How to Use

When a concept first appears in the manuscript:

1. Record its name
2. Note the location (paragraph reference)
3. Describe how it enters (through what behavior, situation, or observation)

When the concept reappears:

1. Note the new location
2. Describe what new weight it carries
3. Ensure it is not reintroduced as if new

## Principles

- Concepts arrive late — only after being felt through behavior
- One abstraction per paragraph, at most
- Concepts must not carry the paragraph — behavior does
- When a concept returns, it must introduce a new angle, cost, actor, time horizon, or consequence
- If a concept is reintroduced as if new, it is a violation

---

## Verification Checklist

At the end of each editing pass:

- [ ] Review all concepts that appear in the worked section
- [ ] Verify no concept restarts as if new
- [ ] Ensure returning concepts carry accumulated weight
- [ ] Update this log with any new concepts or returns
- [ ] Note any violations found and corrected
