# Book / Manuscript Workflow

Use this only when drafting, rewriting, recomposing, editing, or exporting the manuscript `Human, Not Abstract`.

The manuscript is one expression of the skill's content. Do not let manuscript-specific rules leak into unrelated work.

## Required reading

1. `resources/source/writing_voice/author-voice-guide.md`
2. `resources/source/book_work/writing-working-agreement.md`
3. `resources/context/OPERATING_CONTEXT.md` sections 2, 10, 11, and 12
4. `resources/source/book_work/human.md` when continuing or revising existing manuscript

If editing rather than drafting, also read:

5. `resources/source/book_work/EDITING_CHARTER.md`
6. `resources/source/book_work/PATTERN_LEDGER.md`

## Drafting posture

Write as if the thought is being walked, not presented.

Do not:

- Explain the thesis.
- Announce transitions.
- Summarize where the reader is.
- Turn the manuscript into advice.
- Define enablement directly.
- Create clean chapter introductions or conclusions.

Do:

- Keep behavior before abstraction.
- Let concepts return with more weight.
- Keep chapters as terrain changes.
- Preserve unresolved tension.
- Prefer long, uneven paragraphs.
- Let meaning accumulate.

## When continuing a chapter

1. Locate the last current movement in `human.md` or the provided excerpt.
2. Identify what tension remains unresolved.
3. Continue from that tension without orienting the reader.
4. Avoid “now we move to” or “this brings us to”.
5. Do not name the concept too early.

## When creating a downloadable manuscript

1. Produce one complete file.
2. Do not provide diffs unless asked.
3. Do not omit sections.
4. Keep paragraph labels if the user is still working in labeled draft mode.
5. If the user asked for post-pass state, output only that state.
