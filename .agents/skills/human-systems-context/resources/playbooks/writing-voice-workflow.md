# Writing Voice Workflow

Use this when the user asks for writing that should preserve their voice, but the task is not explicitly manuscript/book work.

## Required reading

1. `resources/source/writing_voice/author-voice-guide.md`
2. `resources/context/OPERATING_CONTEXT.md` sections 1, 2, 16, and 17
3. `resources/source/core_stance/who_am_i.md` when personal/professional voice matters

## Response posture

- Direct, human, practical.
- Reflective without becoming ornate.
- Simple words, complex thought.
- Avoid generic leadership or consultancy language.
- Keep copy-paste outputs usable.
- Use Norwegian only when requested or when the user asks for Norwegian text.

## Important boundary

Do not apply the manuscript-only Writing Working Agreement, Editing Charter, or Pattern Ledger unless the user explicitly asks for manuscript/book prose.

For normal work messages, emails, Teams updates, parent messages, coding prompts, or strategy notes, clarity and usefulness matter more than manuscript voice constraints.

## Common output types

- Peer-to-peer work reply.
- Teams status update.
- Parent/coaching message.
- Meeting agenda.
- Professional outreach.
- Strategy note.
- Coding-agent prompt.

## Quality check

Before returning text, check:

- Would the user actually send this?
- Is it too defensive?
- Is it too formal?
- Did it smuggle in a technical solution the user did not want to force?
- Does it preserve the user's stance without over-explaining it?
