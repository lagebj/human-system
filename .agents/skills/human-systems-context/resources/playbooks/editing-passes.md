# Editing Pass Playbook

Use this only when the user declares Pass A, B, C, D, or E for manuscript work.

## Required reading

1. `resources/source/writing_voice/author-voice-guide.md`
2. `resources/source/book_work/writing-working-agreement.md`
3. `resources/source/book_work/EDITING_CHARTER.md`
4. `resources/source/book_work/PATTERN_LEDGER.md`
5. `resources/source/book_work/human.md` or the provided manuscript excerpt

## Hard rule

Never blend passes.

If the user says `Mode: Pass A`, do Pass A only. If you notice an issue belonging to another pass, log it or defer it.

## Pass A — De-patterning

Goal:

- Break repeated mechanical patterns.
- Preserve or increase experiential length.
- Refactor by expansion and causal embedding.

Failure signs:

- Text becomes cleaner but thinner.
- Reader reaches the idea faster.
- Paragraphs become more summary-like.
- The prose reads like polished explanation.

Required output when not creating a downloadable file:

- Paragraph-level actions.
- Ledger check.
- Concept Return Log check.
- Copy-paste-ready CHANGELOG entry.

## Pass B — Causal Tightening

Goal:

- Strengthen internal cause/effect continuity.
- Ensure earlier movements continue downstream.
- Preserve unresolvedness.

Do not:

- Add new conceptual material.
- Reintroduce banned patterns.
- Simplify for readability.

## Pass C — Language and Load Balancing

Goal:

- Rebalance sentence length and paragraph density.
- Preserve conceptual load.
- Ensure short paragraphs are rare and intentional.

Do not:

- Simplify concepts.
- Reframe arguments.
- Normalize the prose into essay structure.

## Pass D — Continuity and Concept Return

Goal:

- Make concept returns accumulate meaning.
- Prevent later sections from restarting ideas as if new.
- Keep the manuscript moving as one walk.

## Pass E — Final Read Integrity

Goal:

- Detect accidental pattern reintroduction.
- Confirm no pass-specific violations remain.
- Only materially edit when a violation is found.

## Downloadable file mode

If the user explicitly asks for a downloadable file:

- Create one complete file.
- No commentary in the file.
- No diffs.
- No omissions.
- The final chat response should only link the file unless the user asked for explanation.
