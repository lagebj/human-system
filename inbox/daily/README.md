# Daily Notes

**Time-based captures tied to specific dates.**

---

## Purpose

Daily notes are for thoughts, observations, and work that belong to a **specific day**:

- What you worked on in the manuscript
- Conversations that shaped thinking
- Observations tied to dates or events
- Running logs of book progress
- Thoughts that emerged from a particular context

Unlike regular inbox items, daily notes are **anchored in time**.

---

## How to Use

Create a file named with the date:

```
inbox/daily/2026-06-12.md
```

Use ISO date format (YYYY-MM-DD) for consistency and sorting.

---

## Template

```markdown
# 2026-06-12

## Context

[What was happening today? Work, reading, conversations, etc.]

## Captures

### [Thought / Observation / Idea]

[Raw capture - don't edit for clarity]

### [Another Thought]

[Another raw capture]

## Work Done

[If you worked on the manuscript today, note what pass, what sections, what decisions]

## References

- [Books, articles, conversations referenced today]

## Follow-ups

- [ ] [Things to revisit or process later]

---

**Processed:** [ ] Yes / [ ] No
**Date Processed:** [when these notes were sorted]
```

---

## When to Use Daily Notes

- You want to capture thoughts **with context** about when/why they emerged
- You're tracking **work progress** on the manuscript
- You had **conversations or readings** that influenced thinking
- You want a **timeline view** of how the book evolved

## When to Use Regular Inbox

- The thought is **timeless** — doesn't belong to a specific day
- You want to capture something **quickly** without date context
- It's a **reference item** (quote, article, pattern) not tied to when you found it

---

## Processing Daily Notes

Like inbox items, daily notes should be processed:

1. Read through the note
2. Extract items that belong elsewhere:
   - Move substantive thoughts to `inbox/` as standalone files
   - Move work outcomes to `docs/editing-log.md`
   - Move insights to appropriate `resources/source/` locations
3. Keep the daily note as a **record of the day**, even if contents are moved
4. Mark as processed

---

## Principles

- **Low friction** — capture first, structure later
- **Context preserved** — daily notes keep the "why now" of thoughts
- **Timeline memory** — provides a raw chronology of the book's development
- **Process rhythmically** — don't let daily notes accumulate unprocessed
