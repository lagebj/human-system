# Inbox

**Low-friction capture for ideas, observations, and inspiration.**

---

## Purpose

This directory exists to make it **easy to drop things in** without thinking about structure, categorization, or where they belong.

When you:
- Have a thought while reading
- Notice a pattern in your work
- Want to capture a quote or reference
- Need to park an idea before it's lost
- Observe something that might belong in the book

...drop it here. Sort it later.

---

## How to Use

### Quick Capture

Create a new file with a descriptive name:

```
inbox/2026-06-12-enablement-as-absence.md
inbox/observation-platform-team-handover.md
inbox/quote-sennett-on-craft.md
inbox/thought-uncertainty-absorption.md
```

Don't worry about:
- Perfect naming
- Correct categorization
- Complete thoughts
- Polished writing

Just capture.

### File Template

```markdown
# [Title / First Line]

**Captured:** YYYY-MM-DD

**Source:** [where this came from - conversation, book, observation, work, etc.]

**Related to:** [optional - concepts, chapters, or other inbox items]

---

[The raw thought, observation, quote, or idea.]

[Don't edit for clarity. Don't structure. Don't resolve. Just write it down.]

---

**Status:** Raw / To Sort / Moved to [location]

**Sorted:** [date and destination when processed]
```

---

## What Belongs Here

- Raw thoughts and observations
- Quotes from books, articles, conversations
- Patterns noticed in work
- Questions without answers yet
- Fragments that might become something
- Inspiration from other domains
- Things you don't want to lose

## What Does NOT Belong Here

- Edited or polished writing
- Structured arguments
- Content ready for the manuscript
- Decisions already made
- Sorted and filed material

---

## Processing the Inbox

The inbox is not a storage system. It is a **transit zone**.

**Weekly (or when ready):**

1. Read through inbox items
2. For each item, decide:
   - **Move to source** — belongs in `resources/source/` as stance or pattern
   - **Move to context** — distilled operational memory for `resources/context/`
   - **Move to manuscript notes** — belongs in `manuscript/notes/`
   - **Move to manuscript** — ready to be woven into `human.md`
   - **Discard** — no longer relevant or useful
   - **Keep in inbox** — not ready yet, needs more time

3. Update the item's status before moving
4. Delete processed files from inbox

**Rule:** An empty inbox is not the goal. A flowing inbox is.

## Daily Notes (Optional)

For time-based captures, use daily notes:

```
inbox/daily/
└── 2026-06-12.md
```

Daily notes are for:
- Thoughts that belong to a specific day
- Observations tied to dates or events
- Running logs of work on the book
- Session notes

See `inbox/daily/README.md` for the daily notes template.

## Principles

- **Frictionless capture** — if it's hard to drop something in, you won't do it
- **No categorization tax** — don't make decisions about structure at capture time
- **Flow over empty** — the inbox should move, not accumulate forever
- **Raw is welcome** — incomplete, messy, and unresolved are all fine
- **Process rhythmically** — sort on a schedule, not when desperate for space
